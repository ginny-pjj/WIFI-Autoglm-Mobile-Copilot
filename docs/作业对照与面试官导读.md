## 二、30 秒项目介绍（面试可直接背）

> 在 USB 版跑通之后，我做了 WiFi 版，对应 Open-AutoGLM 官方的无线 ADB 远程调试。电脑仍跑 FastAPI 后端，手机通过 `adb connect` 无线连接，App 用电脑局域网 IP 发任务。这样不用一直插 USB 线，但仍在本地环境，比云端版更简单，适合作为作业里「远程调试」能力的展示。

## 三、面试官常问 5 题

**Q1：WiFi 版和 USB 版有什么区别？**

> USB 版用 `adb reverse`，App 填 `127.0.0.1`；WiFi 版用 `adb connect`，App 必须填电脑局域网 IP，不能填 localhost。

**Q2：为什么 WiFi 版不能用 127.0.0.1？**

> 127.0.0.1 是手机自己，不是电脑。WiFi 模式没有 reverse，必须走局域网 IP。

**Q3：第一次怎么连 WiFi ADB？**

> 通常 USB 连一次，执行 `adb tcpip 5555`，再在同一 WiFi 下 `adb connect 手机IP:5555`。

**Q4：这和云端版比呢？**

> WiFi 版 PC 和手机要同网，电脑要开着；云端版后端在服务器，手机通过 Tailscale 跨网连接，电脑可以关。

**Q5：对应官方哪部分文档？**

> Open-AutoGLM README 的「远程调试」章节，包括 `adb connect` 和 `--device-id`。
